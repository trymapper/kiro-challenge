Event Management API Documentation

FastAPIベースのイベント管理システムのRESTful APIドキュメント

概要

このAPIは、イベントの作成、取得、更新、削除（CRUD操作）を提供します。すべてのデータはAWS DynamoDBに保存されます。

ベースURL

https://ezlgcqjw65.execute-api.us-west-2.amazonaws.com/prod/

データモデル

Event

イベントの完全な情報を表すモデル

フィールド 型 必須 説明
eventId string はい イベントの一意識別子（英数字、ハイフン、アンダースコアのみ）
title string はい イベントのタイトル（1-200文字）
description string はい イベントの説明（最大1000文字）
date string はい イベントの日付（YYYY-MM-DD形式）
location string はい イベントの場所（1-200文字）
capacity integer はい イベントの定員（1以上）
organizer string はい イベント主催者（1-100文字）
status string はい イベントのステータス（active, cancelled, completed）

EventUpdate

イベント更新用のモデル（すべてのフィールドはオプション）

フィールド 型 必須 説明
title string いいえ イベントのタイトル（1-200文字）
description string いいえ イベントの説明（最大1000文字）
date string いいえ イベントの日付（YYYY-MM-DD形式）
location string いいえ イベントの場所（1-200文字）
capacity integer いいえ イベントの定員（1以上）
organizer string いいえ イベント主催者（1-100文字）
status string いいえ イベントのステータス（active, cancelled, completed）

エンドポイント

GET /

説明: APIの基本情報を返します

レスポンス:

{
  "message": "Event Management API"
}

GET /health

説明: APIの稼働状態を確認します

レスポンス:

{
  "status": "healthy"
}

GET /events

説明: すべてのイベントを取得します

クエリパラメータ:

  • status (optional): イベントのステータスでフィルタリング（active, cancelled, completed）

レスポンス例:

[
  {
    "eventId": "event-123",
    "title": "Tech Conference 2024",
    "description": "Annual technology conference",
    "date": "2024-12-15",
    "location": "Seattle Convention Center",
    "capacity": 500,
    "organizer": "Tech Corp",
    "status": "active"
  }
]

POST /events

説明: 新しいイベントを作成します

リクエストボディ:

{
  "eventId": "event-123",
  "title": "Tech Conference 2024",
  "description": "Annual technology conference",
  "date": "2024-12-15",
  "location": "Seattle Convention Center",
  "capacity": 500,
  "organizer": "Tech Corp",
  "status": "active"
}

レスポンス: 201 Created - 作成されたイベント情報

GET /events/{event_id}

説明: 特定のイベントを取得します

パスパラメータ:

  • event_id: 取得するイベントのID

レスポンス: 200 OK - イベント情報

エラー: 404 Not Found - イベントが見つからない場合

PUT /events/{event_id}

説明: イベントを更新します

パスパラメータ:

  • event_id: 更新するイベントのID

リクエストボディ例:

{
  "title": "Updated Title",
  "capacity": 600
}

レスポンス: 200 OK - 更新後のイベント情報

エラー: 404 Not Found - イベントが見つからない場合

DELETE /events/{event_id}

説明: イベントを削除します

パスパラメータ:

  • event_id: 削除するイベントのID

レスポンス: 200 OK - 削除成功メッセージ

エラー: 404 Not Found - イベントが見つからない場合

エラーレスポンス

すべてのエラーは以下の形式で返されます：

{
  "detail": "エラーメッセージ"
}

使用例

イベント一覧を取得

curl https://ezlgcqjw65.execute-api.us-west-2.amazonaws.com/prod/events

アクティブなイベントのみ取得

curl https://ezlgcqjw65.execute-api.us-west-2.amazonaws.com/prod/events?status=active

イベントを作成

curl -X POST https://ezlgcqjw65.execute-api.us-west-2.amazonaws.com/prod/events \
  -H "Content-Type: application/json" \
  -d '{
    "eventId": "test-event-1",
    "title": "Test Event",
    "description": "This is a test event",
    "date": "2024-12-20",
    "location": "Test Location",
    "capacity": 100,
    "organizer": "Test Organizer",
    "status": "active"
  }'

イベントを更新

curl -X PUT https://ezlgcqjw65.execute-api.us-west-2.amazonaws.com/prod/events/test-event-1 \
  -H "Content-Type: application/json" \
  -d '{
    "capacity": 150
  }'

イベントを削除

curl -X DELETE https://ezlgcqjw65.execute-api.us-west-2.amazonaws.com/prod/events/test-event-1
